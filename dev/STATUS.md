# Project Status

Last updated: 2026-05-16

## What this project is

A multi-track effort to turn Claude (and other LLMs) into a competent
Blender operator for product visualization and short commercial 3D
video — not full Blender mastery, but the ~25% of Blender surface
that maps to product photography and basic motion graphics.

**Three tracks running independently:**

- **Track A — Foundational knowledge corpus** (this repo). QMD-indexed
  markdown. Closes the "stale bpy API" knowledge gap.
- **Track B — Streaming agent harness** (separate, future repo). Built
  on pi.dev + Gemini 2.5 Flash Live API to close the "modality gap"
  (LLM cannot see viewport continuously).
- **Track C — User personalization layer** (separate sibling repo per
  user). Layered on top of Track A. Captures preferences, workflows,
  custom assets, corrections learned over time.

## Open decisions

1. **GitHub remote for user-layer repos.** Each user manages their own.
   Default is local-only or private GitHub. Not centralized.
2. **Skill distribution.** Skill lives in this repo at
   `skills/blender-expert/`. Users install via symlink to
   `~/.claude/skills/`. README has the snippet. Could later package as
   a Claude Code plugin if usage grows.
3. **Cost ceiling per shot iteration.** Deferred until eval data shows
   actual numbers.
4. **Track B (streaming) commitment.** Hold until Track A eval baseline
   is in. Spike Gemini Live API standalone first (~1 day) before
   committing to harness build (4-6 weeks).
